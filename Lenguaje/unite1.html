Unidad 1 · Análisis semántico

El análisis semántico es una fase del compilador encargada de verificar que las instrucciones del programa tengan sentido lógico y respeten las reglas del lenguaje de programación.

1.1 Árboles de expresiones

Los árboles de expresiones son estructuras utilizadas para representar operaciones matemáticas y lógicas de forma jerárquica. Cada nodo del árbol representa un operador o un operando.

Los operandos normalmente se encuentran en las hojas del árbol, mientras que los operadores aparecen en los nodos internos. Esto permite visualizar claramente el orden en que deben ejecutarse las operaciones.

Los compiladores utilizan árboles de expresiones para analizar expresiones complejas y generar código intermedio de manera organizada. 

// Expresión
a + b * c
// Árbol conceptual
      +
     / \
    a   *
       / \
      b   c
Importancia: permiten identificar la prioridad y el orden correcto de las operaciones.

1.2 Acciones semánticas de un analizador sintáctico

Las acciones semánticas son operaciones que realiza el compilador durante el análisis sintáctico para validar el significado del programa.

Estas acciones permiten verificar tipos de datos, registrar variables en la tabla de símbolos y generar información necesaria para las etapas posteriores del compilador.

Cada vez que el analizador reconoce una regla gramatical, puede ejecutar una acción semántica asociada. 

// Declaración
int edad;
// Acción semántica conceptual
Insertar "edad" en tabla de símbolos
Asignar tipo entero
Objetivo: comprobar que el programa tenga sentido además de una sintaxis correcta.

1.3 Comprobaciones de tipos en expresiones

La comprobación de tipos consiste en verificar que las operaciones se realicen entre datos compatibles.

El compilador revisa que variables, constantes y expresiones utilicen tipos válidos para evitar errores durante la ejecución del programa.

Si se intenta combinar tipos incompatibles, el compilador genera un error semántico. 

// Correcto
int a = 5;
int b = 3;
int c = a + b;
// Incorrecto
int edad = 20;
string nombre = "Luis";

edad = edad + nombre;
Error semántico: no se puede sumar un entero con una cadena de texto.

1.4 Pila semántica en un analizador sintáctico

La pila semántica es una estructura utilizada por el compilador para almacenar información temporal durante el análisis de expresiones y sentencias.

En ella se guardan operandos, operadores y resultados parciales que ayudan a validar el significado del programa.

Esta pila trabaja junto con el analizador sintáctico para evaluar reglas gramaticales y construir expresiones correctamente. 

// Expresión
a + b * c
// Operaciones en pila
Push a
Push b
Push c
Aplicar *
Aplicar +
Función: ayuda a controlar y evaluar expresiones durante el análisis.

1.5 Esquema de traducción

El esquema de traducción define las acciones que debe realizar el compilador cuando reconoce determinadas estructuras del lenguaje.

Estas acciones pueden generar código intermedio, actualizar la tabla de símbolos o verificar reglas semánticas.

El esquema de traducción relaciona reglas gramaticales con instrucciones que permiten transformar el programa fuente. 

// Expresión
x = a + b
// Traducción conceptual
t1 = a + b
x = t1
Objetivo: convertir estructuras del lenguaje en instrucciones procesables.

1.6 Generación de la tabla de símbolo y tabla de direcciones

La tabla de símbolos es una estructura utilizada para almacenar información sobre variables, constantes, funciones y otros identificadores del programa.

La tabla de direcciones guarda referencias relacionadas con posiciones de memoria utilizadas durante la ejecución.

Ambas estructuras son fundamentales para las fases de análisis semántico, generación de código intermedio y optimización. 

// Declaraciones
int edad;
float promedio;
// Tabla conceptual
edad      entero      1000
promedio  real        1004
Importancia: permiten organizar y localizar los elementos del programa.

1.7 Manejo de errores semánticos

El manejo de errores semánticos consiste en detectar problemas relacionados con el significado del programa durante la compilación.

Estos errores aparecen cuando el código tiene una sintaxis correcta, pero viola reglas semánticas del lenguaje de programación.

El compilador debe identificar estos problemas y mostrar mensajes claros para ayudar al programador a corregirlos. 

// Variable no declarada
total = precio * cantidad;
// Error de tipos
int edad = 20;
edad = "veinte";
Objetivo: detectar errores antes de ejecutar el programa.
