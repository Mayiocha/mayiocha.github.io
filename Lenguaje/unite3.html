Unidad 3 · Optimización

La optimización busca mejorar el código generado por el compilador sin cambiar el resultado del programa. Su propósito es reducir tiempos de ejecución, disminuir el uso de memoria y eliminar instrucciones innecesarias.

3.1 Tipos de optimización

Los tipos de optimización permiten clasificar las mejoras que puede aplicar un compilador al código. Estas optimizaciones pueden realizarse en partes pequeñas del programa, dentro de ciclos, en funciones completas o en secuencias cortas de instrucciones.

El objetivo es obtener un programa más eficiente, conservando siempre el mismo comportamiento lógico del código original.

Idea principal: optimizar significa mejorar el código sin modificar su resultado.

3.1.1 Locales

La optimización local se aplica dentro de un bloque básico, es decir, una secuencia de instrucciones que se ejecuta de forma continua, sin saltos internos.

Este tipo de optimización permite eliminar instrucciones repetidas, simplificar expresiones y quitar código que no afecta el resultado final. 

// Antes
x = 5 + 3;
y = x + 0;
// Después
x = 8;
y = x;
Ejemplo: se simplifican operaciones dentro de una misma sección del programa.

3.1.2 Ciclos

La optimización de ciclos se enfoca en mejorar instrucciones que se repiten varias veces durante la ejecución. Debido a que los ciclos pueden ejecutarse muchas veces, pequeñas mejoras dentro de ellos pueden producir grandes beneficios.

Una técnica común consiste en mover fuera del ciclo las operaciones que no cambian durante cada iteración. 

// Antes
for (i = 0; i < n; i++) {
    x = a * b;
    y[i] = x + i;
}
// Después
x = a * b;

for (i = 0; i < n; i++) {
    y[i] = x + i;
}
Mejora: la multiplicación se realiza una sola vez en lugar de repetirse en cada vuelta.

3.1.3 Globales

La optimización global analiza partes más amplias del programa, como funciones completas o varios bloques conectados entre sí.

Este tipo de optimización puede detectar valores que se repiten en diferentes secciones, eliminar código innecesario y mejorar el flujo general del programa. 

// Antes
a = b + c;
d = b + c;
e = d * 2;
// Después
a = b + c;
d = a;
e = d * 2;
Ventaja: permite reutilizar resultados y evitar cálculos repetidos en distintas partes del programa.

3.1.4 De mirilla

La optimización de mirilla revisa pequeñas secuencias de instrucciones consecutivas para reemplazarlas por versiones más cortas o eficientes.

Se llama de mirilla porque el compilador observa una pequeña “ventana” del código y busca patrones que puedan simplificarse. 

// Antes
MOV A, B
MOV B, A
// Después
MOV A, B
Objetivo: eliminar instrucciones redundantes en secuencias pequeñas.

3.2 Costos

Los costos permiten medir qué tan eficiente es un programa o una instrucción. En compiladores, se consideran aspectos como tiempo de ejecución, uso de memoria, cantidad de registros utilizados y número de instrucciones generadas.

Analizar los costos ayuda a decidir qué versión del código conviene generar para obtener mejor rendimiento.

Idea principal: el costo indica los recursos necesarios para ejecutar un programa.

3.2.1 Costo de ejecución

El costo de ejecución se refiere a los recursos que consume un programa mientras se ejecuta. Entre ellos se encuentran la memoria, los registros y la pila.

  • Memoria: espacio utilizado para variables, arreglos e instrucciones.
  • Registros: almacenamiento rápido dentro del procesador.
  • Pila: estructura usada para llamadas a funciones y datos temporales.
// Código con mayor costo
x = a + b;
y = a + b;
z = y * 2;
// Código con menor costo
x = a + b;
y = x;
z = y * 2;
Mejora: se reduce el número de operaciones repetidas.

3.2.2 Criterios para mejorar el código

Para mejorar el código, el compilador utiliza criterios que ayudan a decidir qué instrucciones deben cambiarse, eliminarse o reorganizarse.

Estos criterios buscan que el programa sea más rápido, use menos memoria y genere instrucciones más simples para la máquina.

  • Eliminar operaciones innecesarias.
  • Evitar cálculos repetidos.
  • Reducir accesos a memoria.
  • Utilizar registros de forma eficiente.
  • Simplificar expresiones aritméticas.
Propósito: obtener un código más limpio, rápido y eficiente.

3.2.3 Herramientas para el análisis del flujo de datos

El análisis del flujo de datos estudia cómo se mueven los valores dentro de un programa. Permite saber dónde se define una variable, dónde se usa y si su valor sigue siendo necesario.

Este análisis es útil para detectar código muerto, variables no utilizadas, expresiones repetidas y oportunidades de optimización.

  • Grafos de flujo: representan los caminos posibles del programa.
  • Bloques básicos: agrupan instrucciones que se ejecutan juntas.
  • Análisis de variables vivas: identifica variables que aún serán usadas.
  • Análisis de definiciones: muestra dónde se asignan valores a variables.
// Ejemplo
x = 10;
y = x + 2;
x = 20;
print(y);
Observación: el análisis permite saber qué valores siguen siendo importantes en el programa.
