Unidad 2 · Generación de código intermedio

La generación de código intermedio transforma el programa fuente en una representación más sencilla de analizar y modificar. Esta etapa facilita la optimización y sirve como puente entre el análisis del programa y la generación de código objeto.

2.1 Notaciones

Las notaciones son formas distintas de representar expresiones matemáticas y lógicas. Son utilizadas por los compiladores para organizar operaciones y facilitar la traducción del programa a código intermedio.

Dependiendo de la posición del operador respecto a los operandos, una expresión puede escribirse en forma prefija, infija o postfija.

Importancia: las notaciones ayudan a definir claramente el orden de evaluación de una expresión.

2.1.1 Prefija

En la notación prefija el operador se coloca antes de los operandos. Esta forma elimina la necesidad de usar paréntesis para indicar prioridades entre operaciones.

La notación prefija es utilizada en compiladores porque facilita el análisis de expresiones y simplifica algunos procesos de traducción a código intermedio.

En esta notación, primero aparece la operación principal y después los valores sobre los que se aplicará dicha operación. 

// Expresión infija
A + B

// Notación prefija
+ A B
// Expresión infija
(A + B) * C

// Notación prefija
* + A B C
Interpretación: primero se realiza la suma A + B y posteriormente el resultado se multiplica por C.

2.1.2 Infija

La notación infija es la forma más común de escribir expresiones matemáticas y operaciones en lenguajes de programación. En ella, el operador se coloca entre los operandos.

Esta notación es fácil de entender para las personas, aunque requiere reglas de prioridad y uso de paréntesis para evitar ambigüedades.

Los compiladores suelen transformar expresiones infijas en otras representaciones para facilitar el análisis y la generación de código. 

// Ejemplos infijos
A + B
X * Y
C - D
// Expresión compleja
(A + B) * (C - D)
Uso: es la notación utilizada normalmente en matemáticas y programación.

2.1.3 Postfija

En la notación postfija el operador se coloca después de los operandos. También se conoce como notación polaca inversa.

Esta representación facilita la evaluación de expresiones utilizando estructuras tipo pila, por lo que es muy utilizada en compiladores e intérpretes.

Una ventaja importante es que no necesita paréntesis para indicar el orden de evaluación. 

// Expresión infija
A + B

// Notación postfija
A B +
// Expresión infija
(A + B) * C

// Notación postfija
A B + C *
Ventaja: simplifica el procesamiento de expresiones aritméticas.

2.2 Representaciones de código intermedio

Las representaciones de código intermedio son estructuras utilizadas por el compilador para trabajar con el programa antes de producir el código final.

Estas representaciones facilitan el análisis, la optimización y la traducción del programa a lenguaje máquina.

Objetivo: simplificar el proceso de traducción entre el lenguaje fuente y el código objeto.

2.2.1 Notación Polaca

La notación polaca coloca primero el operador y después los operandos. Esta forma de representación evita ambigüedades en expresiones matemáticas.

En compiladores ayuda a representar expresiones de manera estructurada y sencilla. 

// Expresión infija
(A + B) * C

// Notación polaca
* + A B C
// Otro ejemplo
(A - B) / D

// Notación polaca
/ - A B D
Importancia: facilita la representación estructurada de operaciones complejas.

2.2.2 Código P

El Código P es una representación intermedia diseñada para ejecutarse sobre una máquina virtual. Esto permite que el programa sea portable entre distintas plataformas.

Las instrucciones del Código P son más simples que las instrucciones reales del procesador y sirven como paso intermedio antes de generar el código final. 

// Ejemplo conceptual
LDC 5
LDC 3
ADI
STO resultado
// Otro ejemplo
LOD A
LOD B
MPI
STO total
Uso: permite ejecutar programas mediante una máquina virtual.

2.2.3 Tripos

Los triples son una representación de código intermedio que utiliza tres campos: operador, argumento uno y argumento dos. A diferencia de los cuádruplos, no usan un campo separado para guardar el resultado.

El resultado de cada operación se identifica por el número o posición de la instrucción. Esto permite hacer referencia a resultados anteriores sin crear tantas variables temporales. 

// Expresión original
x = a + b * c

// Representación en triples
0: (*, b, c)
1: (+, a, 0)
2: (=, 1, x)
// Otro ejemplo
y = (m - n) / p

// Triples
0: (-, m, n)
1: (/, 0, p)
2: (=, 1, y)
Característica: los triples usan referencias por posición para reutilizar resultados.

2.2.4 Cuádruplos

Los cuádruplos representan una instrucción mediante cuatro elementos: operador, argumento uno, argumento dos y resultado. Son muy utilizados porque permiten ver claramente dónde se guarda cada operación.

Esta representación facilita la optimización, ya que cada resultado temporal tiene un nombre propio y puede analizarse con mayor facilidad. 

// Formato general
(operador, argumento1, argumento2, resultado)

// Ejemplo
(*, b, c, t1)
(+, a, t1, t2)
(=, t2, -, x)
// Otro ejemplo
(-, m, n, t1)
(/, t1, p, t2)
(=, t2, -, y)
Ventaja: son fáciles de leer, analizar y transformar durante la optimización.

2.3 Esquema de generación

El esquema de generación describe cómo el compilador produce código intermedio a partir de las diferentes partes de un programa. Cada construcción del lenguaje necesita una forma específica de traducción.

Este esquema permite convertir variables, constantes, expresiones, asignaciones, instrucciones de control, funciones y estructuras en instrucciones intermedias.

Objetivo: traducir cada elemento del lenguaje fuente a una representación intermedia clara.

2.3.1 Variables y constantes

Las variables son espacios de memoria cuyo valor puede cambiar durante la ejecución del programa. Las constantes, en cambio, mantienen el mismo valor durante todo el proceso.

Durante la generación de código intermedio, el compilador registra estos elementos en la tabla de símbolos y los utiliza para generar instrucciones correctas. 

// Variables
int edad = 20;
float promedio = 9.5;

// Constante
const PI = 3.1416;
// Código intermedio conceptual
edad = 20
promedio = 9.5
PI = 3.1416

2.3.2 Expresiones

Las expresiones combinan variables, constantes y operadores para producir un valor. Pueden incluir operaciones aritméticas, relacionales o lógicas.

En código intermedio, una expresión compleja se divide en instrucciones simples para facilitar su análisis y posterior optimización. 

// Expresión original
x = a + b * c

// Código intermedio
t1 = b * c
t2 = a + t1
x = t2
// Otro ejemplo
z = (a - b) + (c * d)

// Código intermedio
t1 = a - b
t2 = c * d
t3 = t1 + t2
z = t3

2.3.3 Instrucción de asignación

La instrucción de asignación permite guardar un valor dentro de una variable. En los lenguajes de programación se representa comúnmente con el signo igual.

Cuando la asignación contiene una expresión compleja, el compilador la descompone en varias instrucciones intermedias más simples. 

// Asignación original
resultado = (a + b) / c

// Código intermedio
t1 = a + b
t2 = t1 / c
resultado = t2
// Otro ejemplo
total = precio * cantidad + impuesto

// Código intermedio
t1 = precio * cantidad
t2 = t1 + impuesto
total = t2

2.3.4 Instrucciones de control

Las instrucciones de control modifican el flujo normal de ejecución de un programa. Entre ellas se encuentran las condiciones, los ciclos y los saltos.

En código intermedio se representan mediante etiquetas y saltos condicionales o incondicionales. 

// Condición original
if (a > b) {
    x = a;
} else {
    x = b;
}

// Código intermedio
if a > b goto L1
x = b
goto L2
L1: x = a
L2:
// Ciclo original
while (i < 5) {
    i = i + 1;
}

// Código intermedio
L1: if i >= 5 goto L2
t1 = i + 1
i = t1
goto L1
L2:

2.3.5 Funciones

Las funciones son bloques de instrucciones que realizan una tarea específica. Pueden recibir parámetros, ejecutar operaciones y devolver un resultado.

En código intermedio, las funciones se representan mediante instrucciones para pasar parámetros, realizar llamadas y recuperar valores de retorno. 

// Llamada a función
resultado = suma(a, b)

// Código intermedio
param a
param b
call suma, 2
resultado = return
// Otro ejemplo
area = calcularArea(base, altura)

// Código intermedio
param base
param altura
call calcularArea, 2
area = return

2.3.6 Estructuras

Las estructuras permiten agrupar varios datos relacionados bajo un mismo nombre. Cada elemento dentro de una estructura se conoce como campo o atributo.

Durante la generación de código intermedio, el compilador calcula cómo acceder a cada campo de la estructura y cómo almacenar sus valores. 

// Estructura conceptual
persona.nombre = "Luis";
persona.edad = 21;
// Código intermedio conceptual
persona.nombre = "Luis"
persona.edad = 21
t1 = persona.edad
Uso: las estructuras permiten organizar datos relacionados dentro de un programa.
