Unidad 4 · Generación de código objeto

La generación de código objeto transforma el código intermedio en instrucciones que pueden ser entendidas directamente por la computadora. En esta fase se trabaja con registros, lenguaje ensamblador, lenguaje máquina y administración de memoria.

4.1 Registros

Los registros son pequeñas unidades de almacenamiento ubicadas dentro del procesador. Se utilizan para guardar datos temporales y resultados de operaciones mientras el programa se está ejecutando.

Los registros son mucho más rápidos que la memoria RAM, por eso el compilador intenta utilizarlos para acelerar las operaciones matemáticas y lógicas.

Existen diferentes tipos de registros dependiendo de la arquitectura del procesador, como registros de propósito general, registros de control y registros de direcciones. 

// Ejemplo conceptual
LOAD R1, A
LOAD R2, B
ADD R3, R1, R2
STORE C, R3
// Otro ejemplo
MOV AX, 10
MOV BX, 5
SUB AX, BX
Resultado: las operaciones se ejecutan utilizando registros internos del procesador.

4.2 Lenguaje ensamblador

El lenguaje ensamblador es una representación simbólica del lenguaje máquina. Utiliza instrucciones fáciles de identificar para representar operaciones que el procesador puede ejecutar.

Cada instrucción ensambladora corresponde casi directamente a una instrucción binaria del procesador.

Este lenguaje permite tener un mayor control sobre el funcionamiento interno de la computadora y es utilizado en sistemas operativos y programas de bajo nivel. 

// Código ensamblador
MOV AX, 5
MOV BX, 3
ADD AX, BX
// Comparación
CMP AX, BX
JE iguales
Interpretación: las instrucciones permiten mover datos y realizar operaciones directamente en el procesador.

4.3 Lenguaje máquina

El lenguaje máquina está formado por secuencias binarias compuestas por unos y ceros. Estas instrucciones son entendidas directamente por el hardware sin necesidad de traducción.

Cada procesador tiene su propio conjunto de instrucciones, por lo que un programa compilado para una arquitectura específica puede no funcionar en otra diferente.

Aunque el lenguaje máquina es difícil de leer para las personas, representa la forma final en que un programa es ejecutado por la computadora. 

// Representación conceptual
10110000 00000101
00000100 00000011
// Otra instrucción
11100010 00000001
Importancia: es el lenguaje ejecutado directamente por el procesador.

4.4 Administración de memoria

La administración de memoria organiza la forma en que se almacenan variables, instrucciones y datos temporales durante la ejecución del programa.

El compilador debe decidir cómo reservar memoria para variables locales, globales y estructuras necesarias para el funcionamiento del programa.

Una administración eficiente de memoria ayuda a mejorar el rendimiento del sistema y evita errores relacionados con accesos inválidos o pérdida de información.

  • Asignación de memoria para variables.
  • Uso de pila para llamadas a funciones.
  • Manejo de direcciones de memoria.
  • Control de espacios temporales.
// Ejemplo conceptual
PUSH A
PUSH B
CALL sumar
POP resultado
Importancia: permite utilizar los recursos de memoria de manera ordenada y eficiente.
